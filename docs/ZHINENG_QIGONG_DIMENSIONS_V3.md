## 一、文档概述

### 1.1 制定背景

本维度体系基于以下基础资料制定：
- 《智能气功科学概论》《混元整体理论》等核心教材
- 《智能气功功法学》《气功要旨》等功法专著
- 多位专家对智能气功资料分类的评审意见
- 智能气功科学体系框架（基础理论-技术理论-应用理论三层结构）

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **不浪费算力** | 合并冗余维度，减少无效字段，优化查询效率 |
| **不丢失精度** | 保留核心分类信息，支持细粒度检索 |
| **体系完整** | 覆盖智能气功科学的完整理论体系 |
| **可扩展性** | 预留扩展维度，支持未来需求 |

## 二、维度体系详解

### 2.1 A类：内容维度 (4个)

#### 维度1：理论体系归属 (theory_system)

**说明**：标识资料所属的理论体系，是智能气功区别于其他气功流派的根本特征。

| 一级选项 | 二级选项 | 说明 |
|---------|---------|------|
| **混元整体理论** | 混元气理论 | 万物混元气、人的混元气 |
| | 意元体理论 | 意识活动的物质基础 |
| | 意识论 | 意识活动的规律与作用 |
| | 道德论 | 道德与生命优化的关系 |
| | 优化生命论 | 生命优化的理论基础 |
| | 混元医疗观 | 气功医学理论基础 |
| **传统理论借鉴** | 阴阳五行 | 传统中医理论 |
| | 藏象经络 | 经络理论与脏象学说 |
| | 儒释道武 | 传统文化的融合 |
| **现代科学结合** | 生理学 | 现代生理学基础 |
| | 解剖学 | 现代解剖学基础 |
| | 心理学 | 心理学相关内容 |
| | 现代科研 | 科学研究方法 |

#### 维度2：内容主题 (content_topic)

**说明**：描述资料的具体内容主题。

| 一级选项 | 二级选项 | 说明 |
|---------|---------|------|
| **理论类** | 基础理论 | 混元论、整体论等基础内容 |
| | 深层理论 | 混元气、意元体等深层内容 |
| | 方法论 | 内求法、三传并用等方法 |
| | 功理 | 功法原理与机理 |
| **功法类** | 动功 | 六步动功相关内容 |
| | 静功 | 坐功、站功、卧功 |
| | 静动功 | 自发功相关内容 |
| | 组场 | 组场技术与应用 |
| **应用类** | 医学应用 | 第四医学、气功诊断治疗 |
| | 农业应用 | 气功农业增产实验 |
| | 工业应用 | 气功工业提质应用 |
| | 教育应用 | 智能开发、学习提升 |
| | 体育应用 | 体育领域的应用 |
| | 文艺应用 | 文艺创作与表演 |
| **综合类** | 科研研究 | 科学研究工作 |
| | 历史 | 发展历史与重要事件 |
| | 人物 | 相关人物介绍 |
| | 组织 | 组织建设与管理 |
| | 交流 | 对外交流与合作 |

#### 维度3：功法体系 (gongfa_system)

**说明**：按智能气功三阶段六步功法体系分类。

| 阶段 | 功法名称 | 说明 |
|------|---------|------|

| **内混元(中级)** | 形神庄(经络混元) | 气达皮肉筋脉骨 |
| | 五元庄(脏真混元) | 气达五脏六腑 |
| **中混元(高级)** | 中脉混元功 | 打通中脉 |
| | 中线混元功 | 凝炼中线 |
| | 混化归元功 | 天人合一、返本归元 |
| **静功类** | 坐功 | 静坐功法 |
| | 站功 | 站式静功 |
| | 卧功 | 卧式静功 |
| **静动功类** | 自发功 | 需强辅导防偏差 |

## 三、数据模型设计

### 3.1 数据库表结构

```python
# 智能气功资料元数据表
class ZhinengQigongMetadata(Base):
    """智能气功资料元数据 - 13核心维度 + 2扩展维度"""
    __tablename__ = "zhineng_qigong_metadata"

    id = Column(Integer, primary_key=True)
    resource_id = Column(Integer, ForeignKey('resources.id'))
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)

    # ===== A类：内容维度 =====
    theory_system = Column(String(50))      # 1. 理论体系
    theory_subsystem = Column(String(50))   #    子理论

    content_topic = Column(String(50))      # 2. 内容主题(一级)
    content_subtopic = Column(String(50))   #    二级主题

    gongfa_stage = Column(String(20))       # 3. 功法阶段
    gongfa_method = Column(String(50))      #    具体功法

    content_depth = Column(String(20))      # 4. 内容深度

    # ===== B类：情境维度 =====
    timeline_period = Column(String(50))    # 5. 时间段
    timeline_event = Column(String(100))    #    重大事件

    location_institution = Column(String(100))  # 6. 机构
    location_region = Column(String(50))        #    区域
    location_city = Column(String(50))          #    城市

    teaching_level = Column(String(50))     # 7. 教学层次 [合并课程级别+受众]

    presentation_form = Column(String(50))  # 8a. 内容形态
    course_type = Column(String(50))        # 8b. 教学组织
    activity_type = Column(String(50))      # 8c. 活动性质

    # ===== C类：来源维度 =====
    speaker = Column(String(100))           # 9. 主讲/作者

    source_format = Column(String(50))      # 10a. 整理方式
    source_authority = Column(String(50))   # 10b. 权威等级
    source_publication = Column(String(50)) # 10c. 出版状态

    # ===== D类：技术维度 =====
    media_format = Column(String(50))       # 11. 存在形式
    media_language = Column(String(20))     #     语言子属性

    tech_carrier = Column(String(50))       # 12a. 载体
    tech_recording = Column(String(50))     # 12b. 收录方式

    status_completeness = Column(String(20))  # 13a. 完整性
    status_quality = Column(String(20))       # 13b. 质量
    status_processing = Column(String(20))    # 13c. 处理状态
    status_publishing = Column(String(20))    # 13d. 发布状态

    # ===== E类：扩展维度(可选) =====
    application_effects = Column(JSONB)     # 14. 应用成效
    related_resources = Column(JSONB)       # 15. 关联网络

    # 索引
    __table_args__ = (
        Index('ix_theory_system', 'theory_system'),
        Index('ix_content_topic', 'content_topic'),
        Index('ix_gongfa_method', 'gongfa_method'),
        Index('ix_timeline_period', 'timeline_period'),
        Index('ix_speaker', 'speaker'),
        Index('ix_teaching_level', 'teaching_level'),
    )
```

### 3.2 关联表设计

```python
# 资料关联关系表
class ResourceRelation(Base):
    """资料间的关联关系"""
    __tablename__ = "resource_relations"

    id = Column(Integer, primary_key=True)
    source_id = Column(Integer, ForeignKey('resources.id'))
    target_id = Column(Integer, ForeignKey('resources.id'))

    relation_type = Column(String(50))  # 引用/补充/前导/后续/版本

    created_at = Column(DateTime, default=datetime.utcnow)

    __table_args__ = (
        UniqueConstraint('source_id', 'target_id', 'relation_type',
                        name='uix_source_target_relation'),
    )
```

### 5.2 分类一致性保障

1. **词表约束**：使用受控词表，禁止自由输入

3. **审核流程**：新增资料需经审核确认分类
4. **定期校验**：定期检查分类一致性
