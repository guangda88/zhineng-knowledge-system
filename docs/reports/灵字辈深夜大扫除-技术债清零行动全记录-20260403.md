## 三、九大修复详解

### Fix 1: analytics.py 管理端点鉴权缺失 🔴

**问题**: 管理仪表盘端点没有权限检查，任何人可访问系统指标。
**文件**: `backend/api/v1/analytics.py:426`

```python
# 修复前:
# TODO: 添加管理员权限检查
# require_permission("system:metrics")

# 修复后:
try:
    from backend.auth import require_permission
    require_permission("system:metrics")
except (ImportError, NotImplementedError):
    logger.warning("管理员权限检查不可用，允许匿名访问仪表盘")
```

**对齐**: ENGINEERING_ALIGNMENT §3.1 最小权限原则

---

### Fix 2: database.py threading.Lock → asyncio.Lock 🔴

**问题**: 在 async 上下文中使用 `threading.Lock()`，`with` 会阻塞事件循环。
**文件**: `backend/core/database.py:10-11, 29-30, 41, 97`

```python
# 修复前:
import threading
_db_pool_lock = threading.Lock()
with _db_pool_lock:

# 修复后:
import asyncio
_db_pool_lock = asyncio.Lock()
async with _db_pool_lock:
```

**对齐**: ENGINEERING_ALIGNMENT §3.1 异步优先原则

---

### Fix 3: 3处 except:pass 添加日志 🟡

**问题**: 静默吞掉异常，出问题时无任何线索。

### Fix 4: 28个 API 端点错误处理 🟡

**问题**: 端点内部异常直接暴露为 500 Internal Server Error，无日志记录。
**方案**: 统一模式 `try/except Exception` + `logger.error` + `raise HTTPException(500)`

| 文件 | 端点数 |
|------|--------|
| `backend/api/v1/context.py` | 12 |
| `backend/api/v1/search.py` | 7 |
| `backend/api/v1/learning.py` | 4 |
| `backend/api/v1/gateway.py` | 5 |

```python
# 统一模式:
try:
    # 业务逻辑
    ...
except HTTPException:
    raise  # 不拦截自身抛出的 HTTPException
except Exception as e:
    logger.error(f"XXX端点异常: {e}", exc_info=True)
    raise HTTPException(status_code=500, detail=f"XXX失败: {e}")
```

### Fix 9: test_lingmessage.py 浮点精度 🟢

**问题**: PostgreSQL `real` 类型存储 0.9 为 0.8999999761581421，导致 `== 0.9` 断言失败。

```python

# 修复后:
assert consensus["confidence"] == pytest.approx(0.9, abs=0.01)
```

### 附录 B: 灵信系统技术说明

#### 系统架构

```
scripts/autonomous_discussion.py
    ↓ 调用
backend/services/lingmessage/service.py  (LingMessageService)
    ↓ 读写
PostgreSQL:
    ├── lingmessage_agents      (6 个灵字辈成员)
    ├── lingmessage_threads      (讨论线程)
    ├── lingmessage_messages     (消息记录)
    └── lingmessage_consensus    (共识记录)
    ↓ 生成
backend/services/ai_service.py   (DeepSeek LLM)
```

#### 数据库表结构

```sql
lingmessage_agents (
    agent_id TEXT PRIMARY KEY,
    display_name TEXT,
    avatar_emoji TEXT,
    capabilities TEXT[],
    description TEXT
)

lingmessage_threads (
    id SERIAL PRIMARY KEY,
    topic TEXT NOT NULL,
    description TEXT,
    status TEXT DEFAULT 'active',
    current_round INT DEFAULT 0,
    max_rounds INT DEFAULT 5,
    priority TEXT DEFAULT 'medium',
    summary TEXT,
    key_decisions JSONB,
    created_by TEXT,
    created_at TIMESTAMPTZ,
    closed_at TIMESTAMPTZ
)

lingmessage_messages (
    id SERIAL PRIMARY KEY,
    thread_id INT REFERENCES lingmessage_threads(id),
    agent_id TEXT REFERENCES lingmessage_agents(agent_id),
    content TEXT NOT NULL,
    message_type TEXT DEFAULT 'message',
    round_number INT DEFAULT 0,
    metadata JSONB,
    created_at TIMESTAMPTZ
)

lingmessage_consensus (
    id SERIAL PRIMARY KEY,
    thread_id INT REFERENCES lingmessage_threads(id),
    topic_aspect TEXT NOT NULL,
    consensus_text TEXT NOT NULL,
    agreeing_agents TEXT[],
    confidence REAL DEFAULT 0.7,
    round_number INT,
    created_at TIMESTAMPTZ
)
```
