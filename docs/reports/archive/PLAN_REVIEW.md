## 技术栈审查

### 建议保留 ✅

| 技术 | 理由 |
|------|------|
| FastAPI | 简单高效，异步支持好 |
| PostgreSQL + pgvector | 一体化，运维简单 |
| Redis (已有) | 缓存必需 |
| Nginx | 反向代理标准 |

### 建议删除 ❌

| 技术 | 理由 |
|------|------|
| Node.js (前端) | 纯 HTML/JS 足够 |
| 复杂前端框架 | 单人维护成本高 |
| Celery (任务队列) | MVP 不需要 |
| MinIO (存储) | 对象存储非必需 |

### 建议延迟 ⏳

| 技术 | 理由 |
|------|------|
| Neo4j (图谱) | GraphRAG 可以后期添加 |
| Grafana | 监控可以简化 |
| API Gateway | 单服务不需要 |
