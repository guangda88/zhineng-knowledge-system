## 2. 项目结构对齐

### 2.1 规划 vs 实际结构

#### 规划要求 (单文件优先)
```
backend/
├── main.py             # 主入口 (单文件优先)
├── config.py           # 配置管理
├── models.py           # 数据模型
```

#### 实际结构 (模块化)
```
backend/
├── main.py             # 主入口 (1052行，偏长)
├── config.py           # 配置管理
├── models.py           # 数据模型
├── api/                # API 子模块
├── services/           # 业务服务
│   ├── retrieval/      # 检索
│   ├── reasoning/      # 推理
│   └── rag/            # RAG
├── auth/               # 认证授权
├── cache/              # 缓存
├── monitoring/         # 监控
├── domains/            # 领域系统
└── gateway/            # API网关
```

### 2.2 对齐分析

| 规则要求 | 实际情况 | 符合度 | 说明 |
|----------|----------|--------|------|
| 单文件优先 | 模块化架构 | ❌ 不符合 | 架构更复杂 |
| 类型注解 | 部分有注解 | ⚠️ 部分符合 | 需加强 |
| 文档字符串 | 主要函数有 | ⚠️ 部分符合 | 缺Raises |
| 异步优先 | 全部使用async | ✅ 符合 | 良好 |
